About Me

Born and raised in the mountains of Colorado, I came to appreciate the beauty found in the mountains that surrounded me and developed a sense of adventure and curiosity from a young age. That curiosity grew as I traveled to many places with my parents. These experiences instilled in me me a fresh perspective of what surrounded me and a desire to learn about the new places, peoples, cultures, and landscapes I encountered. While deciding on a major for my undergraduate degree, I attended a study abroad to Tanzania with the Geography Department. That experience enriched my perspective and increased my desire to learn even more. Upon returning home, I declared Geography as a major and have never looked back since. I had a goal to find something that I enjoyed doing to pursue for a career. With GIS, I achieved that goal. I love what I do from getting to create and design maps on many different topics to visualizing and solving problems from a spatial perspective. I feel that I continue to learn something new every day.

My focus in my work is to help others see the world from a new perspective, inspire others to learn more about the world around them, and tell the stories that surround us. I do this by creating maps, dashboards, applications, and presentations using tools such as ArcGIS Pro, Adobe Illustrator, ArcGIS API for JavaScript, and StoryMaps for ArcGIS. Additionally, I strive to keep learning and expanding my skills to help accomplish my goal. As such, I am currently pursuing a master's degree in GIS Development to develop and enhance my coding and development skills. Discovering GIS has been a blessing I never knew I needed, and with it, I have found my true passion in life.

Highlighted Work

Academic Work
Tanzania Reference Map

I designed the above reference map of Tanzania using ArcGIS Pro as part of a cartography class while earning my undergraduate degree at Brigham Young University. This map represents my first cartographic design project and it provided a great introduction to cartographic design principles, including generalization and labeling.

Realistic Relief Map

I created the above realistic relief map of a hypothetic bike route during the Graphic Design in Cartography course at University of Wisconsin-Madison (UW-Madison). Using both Adobe Photoshop and Illustrator, I focused on visual hierarchy to make the bike route stand out with the reference information fading into the background (figure-ground concept). The project was a great introduction to using Adobe Photoshop and Illustrator in the map-making process.

Utah Soda Shops

I created this interactive web application as part of a project for the Interactive Cartography and Visualization course at UW-Madison. It incoporated Leaflet and Data-Driven Documents (D3) JavaScript libraries. The map was interactive and both the map and pie chart updated to reflect client-side filtering of the data. The application was one of my first front-end interactive web applications using JavaScript.

Grand Teton Lodge Company Guests

I designed the above web application while taking the Spatial Web & Mobile Programming course at UW-Madison. The application, designed for the Grand Teton Lodge Company managers and marketing teams, connected to a PostgreSQL database containing all of the guests data and allowed for writing new records into the database, querying the database, and updating records in the database all from the front-end application. The map and pie charts updated to reflect changes made to the database. The application was built primarily using JavaScript for the front-end and Java for the back-end, with the use HTTP GET & POST requests and responses and a Java JSP for communication between. The application was my first that contained both a front-end & back-end and the interactions in between.

National Parks Database

The final project of the Spatial Databases course at UW-Madison, involved designing and implementing a database in order to solve a problem. My group focused on designing a geodatabase to compare which national parks received the most visitation and whether the nearby transportation networks or nearby population areas had any correlation with visitation. The goal was to enhance understanding of the factors that influence national park visitation and aim to mitigate the impacts on national parks that result from increased visitation. The link to my group's final report which illustrates the database design can be found here.

Time & Leaflet

As part of the Interactive Cartography and Visualization course at UW-Madison, we practiced implementing interaction in multiple dimensions, including time. By leveraging the Leaflet JavaScript library, I created an interactive map showing national park visitation numbers that cycled through the years based on user interaction with an operator. This was my first application built with JavaScript.

Coordinated Visualization & D3

As part of the Interactive Cartography and Visualization course at UW-Madison, I created a mapping application using the D3 JavaScript library that implemented the idea of coordinated visualization: interaction with the chart updated the map to reflect the interaction (e.g. selecting a point in the chart highlighted the associated county in the map in red as seen in the image above) and vice versa.

Spatial Analysis using Arcpy

As part of the Capstone in GIS Development course at UW-Madison, we were given a task to design an application that would perform spatial analysis methods based on user input to examine the relationship between cancer rates and nitrate levels in the state of Wisconsin. Building the application required that I utilize what I had learned regarding python from the Introduction to Geocomputing course and spatial analysis from the GIS and Spatial Analysis courses at UW-Madison. For the application, I leveraged the library of Arcpy for the spatial analysis and the library of Tkinter for the GUI. Click here to read the report describing the application and spatial analysis methods used.

Exploring GTNP Mobile-Friendly Application

As part of the Capstone in GIS Development course, we were given a task to design a mobile-friendly application for a park that was cohesive and centered around themes. I designed the application for first-time visitors to Grand Teton National Park and focused on the themes of exploring, learning, safety, and enjoyment. The application also allowed users to contribute to one of the data layers which used API calls for inserting the new data (PUT) and retrieving the updated data (GET). I primarily used Bootstrap to make the application cohesive & responsive and Mapbox for the map & application functionality, as well as the storage of the data. Click here to view the application.

Infographic

I created this infographic as part of Graphic Design in Cartography at UW-Madison. It was my first project in Adobe Illustrator.

Spatial Analysis using Arcpy

I designed this web application as part of the Practicum in GIS Development course at UW-Madison. This project was a culminating final project of the program. I utilized primarily Esri's ArcGIS Maps SDK for JavaScript to design the front-end of the application and calculate consequence of failure scores for pipelines based on spatial relationships to nearby features. I also utilized D3 for charts. The scores were stored in a PostgreSQL database and a web-based controller API written in C# was used for communication between the front-end and back-end of the application.

Split Symbol Map

I utilized ArcGIS Pro and Adobe Illustrator to create the above split symbol map for a presentation comparing 2020 and 2021 reservoir peaks. Since I created the presentation for the board of directors, I needed to design it in a way that presented the information powerfully yet simply. Using the split symbol technique achieved both of those goals as the split symbols made it easy to compare the two years while still conveying the story of the lower reservoirs in 2021.

Tolkien-Themed District Map

At CUWCD, we recently updated the District maps. One of the more interesting ones I got to be involved with was applying a fun Tolkien-style theme to the basic map for the break room. I had to apply various cartographic principles to ensure it matched the theme, while still conveying the intended information.

Automating Workflows with Python

One of the most tedious workflows we had at CUWCD was the process of updating the vector tile packs on a monthly basis. Using existing python scripts and the knowledge I had learned of python, I was able to generate a script that completely automated the process and only required entering in a few parameters in the command line to run, rather than the 1-2 days of manual effort it used to take to complete that process.
