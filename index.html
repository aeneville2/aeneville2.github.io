CUWCD Pipeline COF Scoring Application

Tools: ArcGIS Maps SDK for JavaScript, D3, jQuery UI, Bootstrap, ASP.NET Core (C#)

Course: GEOG 778 - Practicum in GIS Development (UW-Madison)

Project Background: This interactive web application was my culminating final project for the Master's in GIS Development degree from UW-Madison. I designed it for the Central Utah Water Conservancy District (CUWCD) to evaluate the consequence of failure (COF, aka the potential impacts that would occur should the pipeline fail) for their pipelines. COF is scored by evaluating the spatial relationships between a pipeline and surrounding features (e.g., distances to nearby roads). As such, this interactive web map application, is designed to calculate COF scores for CUWCD pipelines using methods that evaluate the spatial relationships between a selected pipeline and nearby features, write those scores to the PostgreSQL database through the use of API calls to a controller-based web API written in C#, and view the scores in tables and charts for comparison to help determine pipeline replacement schedules. Due to privacy constraints regarding the data, the application is located behind a firewall and is unable to be shared here.

Exploring Grand Teton National Park (GTNP)

Tools: Mapbox, Bootstrap, D3, JavaScript

Course: GEOG 777 - Capstone in GIS Development (UW-Madison)

Exploring the Spatial Relationship Between Cancer and Nitrate

Tools: Python, Arcpy, Tkinter

Course: GEOG 777 - Capstone in GIS Development (UW-Madison)

Project Background: As part of the Capstone in GIS Development course at UW-Madison, we were given a task to design an application that would perform spatial analysis methods based on user input to examine the relationship between cancer rates and nitrate levels in the state of Wisconsin. Building the application required that I utilize what I had learned regarding python from the Introduction to Geocomputing (GEOG 378) course and spatial analysis from the GIS and Spatial Analysis (GEOG 579) course at UW-Madison. For the application, I leveraged the library of Arcpy for the spatial analysis and the library of Tkinter for the GUI. The spatial analysis methods of IDW, Zonal Statistics, Joining, and Generalized Linear Regression were used. In the GUI, the user could input different values for the distance decay coefficient used in the IDW step of the analysis. Then, the analysis would be performed and a resulting standard deviation map, linear regression equation, and statistical test values would be output to the GUI for the user to evaluate and save the results.

Grand Teton Lodge Company Guests

Tools: Google Maps, D3, JavaScript, Java, PostgreSQL Database

Course: GEOG 576 - Geospatial Web and Mobile Programming (UW-Madison)

Project Background: The goal of the project was to create a web application with client-server communication to solve a problem. It was my first project involving both client-side and server-side development. I designed it for managers and the marketing teams of the Grand Teton Lodge Company to understand if there were patterns between where a guest originates from and their preferences while staying in the park. The guests data was stored in a table in a PostgreSQL database. From the client, the user can choose to only display guests in the map and charts with certain stay preferences (such as month of stay), add new guests into the database, and update the data for existing guests in the database. Based on the user interactions, HTTP GET & POST requests communicate with a Java JSP that then connects to the database and performs the necessary SQL query before returning the guests data for display in the map and pie charts.

Soda Shops in Utah
